# Project-Use

## 触发条件

开发外使用或维护项目时触发：部署、运行、维护、排查问题、使用功能。

## 流程

1. **读取 ops.md** - 了解使用规则、部署方法、故障排除
2. **按指南操作** - 执行部署/测试/排查/维护
3. **解决问题** - 查看日志、检查配置、测试连接
4. **更新 ops.md** - 记录问题和解决方案

## ops.md 记录规范

- 只记录真实存在的问题和解决方案
- 使用简洁格式：问题/原因/解决
- 不添加装饰性内容、不记录无关信息
- 示例：

```markdown
### 数据库连接失败

**原因**：db_manager.py 未传递 schema 参数
**解决**：init_postgres_db() 新增 schema 参数，从配置库读取 database_schema
```

## 关键原则

- 先读文档，后动手
- 解决问题后更新 ops.md
- 保持文档简洁、准确、实用

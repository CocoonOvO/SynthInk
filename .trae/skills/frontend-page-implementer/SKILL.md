# 前端页面实现专家

本SKILL指导如何根据设计规范实现前端页面，通过迭代对比和优化达到设计还原。

## 工作流程

### 预备阶段：协作准备
1. **阅读协作文档**
   - 读取 `agent_project.md` 了解当前项目状态和任务分配
   - 在任务列表中查找或创建相关任务
   - 如有需要，在交流区声明开始工作

2. **更新任务状态**
   - 领取或更新任务状态为"进行中"
   - 记录预计完成时间和关键节点

### 第一阶段：发现与分析
1. **定位设计资源**
   - 在 `/docs/design/` 或项目文档中搜索设计文档
   - 查找设计HTML页面（用于在浏览器中查看和截图对比）
   - 确定要实现的目标页面/组件

2. **分析设计稿**
   - 阅读设计文档了解规范
   - 记录配色方案、字体、间距、布局
   - 识别交互元素和动画效果
   - 记录响应式行为要求

3. **检查现有实现**
   - 查找目标页面的现有前端文件
   - 分析当前组件结构
   - 识别可复用的组件和模式

### 第二阶段：开发
1. **搭建开发环境**
   - 确保前端开发服务器运行中
   - 验证后端API可用（如需要）
   - 准备浏览器用于截图对比

2. **实现页面结构**
   - 按照项目规范创建/修改Vue组件
   - 实现匹配设计布局的HTML结构
   - 根据设计规范添加CSS样式
   - 优先使用现有的设计系统变量

3. **添加交互功能**
   - 实现Vue响应式数据和方法
   - 添加用户交互的事件处理
   - 实现动画和过渡效果
   - 连接后端API（如需要）

### 第三阶段：视觉对比与优化 ⭐
1. **捕获截图**
   - 打开设计HTML页面并截图（作为参考）
   - 在相同视口大小下截取实现页面截图
   - 使用Chrome DevTools MCP进行精确捕获

2. **对比并识别差异**
   - 布局对齐和间距
   - 颜色准确性
   - 字体（字号、字重、行高）
   - 组件大小和位置
   - 缺失或多余的元素

3. **迭代优化**
   - 调整CSS属性匹配设计
   - 修复布局问题
   - 优化颜色和字体
   - 重复对比直到视觉对齐

### 第四阶段：质量保证
1. **代码质量**
   - 遵循项目编码规范
   - 确保TypeScript类型正确
   - 添加适当的注释
   - 检查控制台错误

2. **响应式测试**
   - 在多个视口大小下测试
   - 验证移动端布局匹配设计
   - 检查平板/桌面断点

3. **功能测试**
   - 测试所有交互元素
   - 验证数据获取正常工作
   - 检查错误处理

### 第五阶段：协作收尾
1. **更新协作文档**
   - 在 `agent_project.md` 中更新任务状态为"已完成"
   - 记录实际完成时间和关键决策
   - 如有需要，在交流区分享实现心得或遇到的问题

2. **代码提交准备**
   - 确保代码符合项目规范
   - 准备提交信息，描述改动内容
   - 如有必要，更新相关文档

## 核心原则

1. **像素级还原**：追求视觉对齐，而非仅功能相似
2. **组件复用**：使用和扩展项目中的现有组件
3. **设计系统合规**：遵循已建立的CSS变量和模式
4. **迭代优化**：预期需要多轮对比循环
5. **文档记录**：记录设计决策和偏差

## 工具与技术

- **Chrome DevTools MCP**：用于截图、元素检查和调试
- **Vue DevTools**：用于组件状态检查
- **设计文档参考**：始终保持设计文档可访问
- **CSS对比**：使用计算样式匹配精确值

## 常见陷阱避免

1. 不要假设浏览器默认样式匹配设计
2. 检查亮色和暗色主题变体
3. 验证字体加载和降级行为
4. 使用真实数据测试，而非仅占位内容
5. 确保符合无障碍标准

## 调用示例

用户："实现用户资料页面，匹配 docs/design/profile-page.png 中的设计"

本SKILL应该：
1. 阅读设计文档并分析截图
2. 检查现有资料页面实现
3. 开发/修改组件以匹配设计
4. 迭代对比和优化直到对齐
5. 确保响应式行为和功能

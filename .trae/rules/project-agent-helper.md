## 面向博客用户agent的mcp
你的项目需要支持mcp以便于后续agent学习
你需要做到：
1. 自行检查和创建当前项目的mcp子项目
2. 使用fastmcp开发维护mcp服务。该mcp服务面向博客用户agent，应当尽可能包含所有可能用到的接口
3. 每次更新项目接口后，都应当检查是否需要更新mcp服务
## 面向项目运维agent的介绍
除了基本的readme，你需要额外在项目下维护一个ops.md文件，用于介绍项目的开发和运维相关内容。包括但不限于：
1. 项目的架构和组件介绍
2. 项目的开发风格和规范
3. 项目的部署和运行环境要求
4. 项目的监控和日志管理
5. 项目的备份和恢复策略
6. 项目的安全和权限管理
你可以为这些内容预留空间，并在后续更新项目时，根据实际情况填写。
在ops.md中，你应当遵循如下规范：
1. 只记录项目中真实存在已经的功能
2. 所有操作中涉及的可变部分（如参数，数据库，用户等），都应当用变量表示，而不是具体的数值或字符串。且你需要说明实际操作如何替换
3. 对于具体操作和代码，你不需要过于详细，只要说明操作的目的和效果即可。请记住，这个文件面向的是agent，他有足够的知识，且会自己阅读代码
4. 你记载的所有操作，都应当是真实存在的，不能只是理论上的，最好只记录你在开发过程中真实用到了的操作

重要：你每次更新项目后都应当检查更新ops
